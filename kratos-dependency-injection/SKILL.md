# Kratos Dependency Injection with Wire

## Overview

This skill provides a guide to using Google's Wire for compile-time dependency injection in a Kratos application. The Kratos project layout is designed around Wire to connect all the components of your application, from data access up to the server transports.

## When to Use

Use this skill when you need to:
- Understand how different layers (`data`, `biz`, `service`) are connected in a Kratos project.
- Add a new dependency (like a database connection or a new repository) to your application.
- Troubleshoot dependency injection errors from Wire.
- Regenerate the `wire_gen.go` file after making changes to your providers.

## Core Concepts

- **Provider**: A normal Go function that creates and returns a value (e.g., `NewGreeterRepo`). Providers can have their own dependencies as parameters.
- **ProviderSet**: A collection of providers that are grouped together for a specific layer or feature (e.g., `data.ProviderSet`).
- **Injector**: A function signature defined in `wire.go` that tells Wire what final object to build (e.g., `*kratos.App`). The function body must be `panic(wire.Build(...))).
- **`wire_gen.go`**: The file auto-generated by Wire that contains the actual dependency injection logic. You should never edit this file manually.

## Core Workflow

1.  **Define Providers:** In each layer (`data`, `biz`, `service`), create constructor functions for your components.

    ```go
    // internal/data/data.go
    // NewGreeterRepo is a provider.
    func NewGreeterRepo(data *Data, logger log.Logger) biz.GreeterRepo {
        return &greeterRepo{data: data, log: log.NewHelper(logger)}
    }
    ```

2.  **Group Providers in a ProviderSet:** In the same package, group your providers into a `ProviderSet`.

    ```go
    // internal/data/data.go
    var ProviderSet = wire.NewSet(NewData, NewGreeterRepo)
    ```

3.  **Build the Injector:** In `cmd/server/wire.go`, use `wire.Build` to assemble all the `ProviderSet`s from your layers.

    ```go
    // cmd/server/wire.go
    //go:build wireinject
    // +build wireinject

    func wireApp(*conf.Server, *conf.Data, log.Logger) (*kratos.App, func(), error) {
        panic(wire.Build(server.ProviderSet, data.ProviderSet, biz.ProviderSet, service.ProviderSet, newApp))
    }
    ```

4.  **Generate the Code:** Run `go generate` from your project's root directory. This will execute the `wire` tool and create or update `cmd/server/wire_gen.go`.

    ```bash
    go generate ./...
    ```

## Common Mistakes

- **Manually Editing `wire_gen.go`:** Any changes made to this file will be overwritten the next time `go generate` is run. Always make changes to your providers or the `wire.go` injector file.
- **Forgetting to Add a New Provider to the `ProviderSet`:** If you create a new component but forget to add its constructor to the `ProviderSet`, Wire will complain about a missing dependency.
- **Interface Mismatches:** If a provider returns a concrete type (e.g., `*greeterRepo`) but another provider depends on an interface (`biz.GreeterRepo`), ensure your constructor function's return type is the interface to satisfy Wire.
- **Circular Dependencies:** Wire will detect circular dependencies at compile time. This usually indicates a design flaw that needs to be resolved by refactoring your components.
